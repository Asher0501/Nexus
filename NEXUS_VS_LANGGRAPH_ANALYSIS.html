Nexus

定位有向图驱动的子进程编排引擎——AI 生成工作流、引擎确定执行
语言Rust (edition 2024) + Python 辅助脚本
图库petgraph::StableDiGraph
节点子进程（stdin JSON → stdout JSON 协议），任何语言
接口CLI + Dashboard HTTP/WS + MCP JSON-RPC
核心创新h_e + g_e 正交边模型；调度/数据流分离；Errors as Routing Paths

LangGraph

定位基于有向状态图的LLM Agent 框架——开发者手写、框架管理状态与持久化
语言Python + TypeScript（双语言）
图模型自研 StateGraph → CompiledStateGraph
节点进程内函数（Python/TS callable），类型安全
接口Python/TS SDK + LangSmith + LangGraph Platform
核心创新Pregel Super-Step；Reducer 状态合并；每步 Checkpoint；HITL interrupt()
核心差异一句话：
Nexus = 子进程级别的编排引擎——任何可执行命令即节点，stdin/stdout JSON 协议通信。面向「AI 生成工作流 → 引擎确定执行」。
LangGraph = 函数级别的 Agent 框架——Python/TS 函数即节点，中心化 State 对象隐式传递。面向「开发者手写工作流 → 框架管理状态」。

h_e —— 分支匹配（纯函数，无记忆）

对每条出边执行三步判定：

// ① 事件类型匹配
if edge.event_type != event { continue; }  // Complete≠Failed → 跳过

// ② exit_reason 过滤器（可选）
if let Some(ref expected) = edge.exit_reason {
    if exit_reason != Some(expected) { continue; }  // "ok"≠"review" → 跳过
}

// ③ 阈值计数器
edge_state.event_count += 1;
if edge_state.event_count < edge.threshold { continue; }
// threshold=3 意味着同一事件类型要发生3次才触发
g_e —— 策略聚合
match edge.strategy {
    Strategy::Any => {
        ready_queue.push_back(edge.to);   // 立即入队下游
    }
    Strategy::All => {
        fan_in_pending[edge.to] -= 1;     // 等待其他上游
        if fan_in_pending[edge.to] == 0 { // 所有上游到齐
            ready_queue.push_back(edge.to);
            // 重置计数器供下一轮使用
            fan_in_pending[edge.to] = all_edge_count;
        }
    }
}
静态边 —— 固定跳转
graph.add_edge("node_a", "node_b")    # A 完成后总是跳转到 B
graph.add_edge(START, "entry")        # 指定入口节点
graph.add_edge("final", END)          # 到达终点
条件边 —— Python 函数动态路由
def router(state: AgentState) -> str:
    if state["confidence"] > 0.8:
        return "auto_approve"
    elif state["confidence"] > 0.3:
        return "human_review"
    return "reject"

graph.add_conditional_edges(
    "classifier", router,
    {"auto_approve": "approve_node",
     "human_review": "review_node",
     "reject": "reject_node"}
)
Command —— 节点内动态路由（推荐）
def smart_node(state: State) -> Command:
    result = analyze(state)
    return Command(
        update={"analysis": result},    # 状态更新
        goto="next_step",               # 路由目标
        graph=Command.PARENT            # 可选：跳转到父图
    )
